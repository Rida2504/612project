# TextWorld VR — WebXR Viewer

Quick-start for rendering trained Gaussian splats in-browser and in Quest 3 VR.

## Local dev (Mac / desktop browser)

```bash
# From the textworld-vr/ root (one level up from this viewer/ dir):
python -m http.server 8000

# Then open:
http://localhost:8000/viewer/
```

Drag & drop any `.ply`, `.splat`, or `.ksplat` file onto the page to load it,
or pick from the scene dropdown (populated from `scenes.json`).

## Quest 3 (LAN, HTTP — dev only)

1. Start the server on your Mac: `python -m http.server 8000`
2. Find your Mac's LAN IP: `ipconfig getifaddr en0`
3. On Quest 3, open the Meta Browser and go to `http://<LAN-IP>:8000/viewer/`
4. Load a scene, then tap **Enter VR**.

WebXR typically requires HTTPS. For LAN testing, Meta Browser has flags to
allow `http://` WebXR; for production use HTTPS (below).

## Quest 3 (public URL, HTTPS — recommended for demo)

Easiest options, zero cost:
- **ngrok:** `ngrok http 8000` → shares an HTTPS URL to your local server.
- **Vercel / Netlify:** `vercel deploy` or `netlify deploy` — drop the
  `viewer/` and `outputs/splats/` dirs into the site root.
- **GitHub Pages:** push the static viewer + splats to a gh-pages branch.

Once live, open the HTTPS URL on Quest 3 browser → tap **Enter VR**.

## Scene list

`scenes.json` drives the scene picker dropdown. Each entry:

```json
{
  "name": "human-readable title",
  "url": "relative/path/to/scene.ply",
  "pipeline": "depth | legacy-v2 | layerpano3d",
  "prompt": "the text prompt used"
}
```

Paths are relative to `viewer/index.html`. Splats generated by the pipeline
live in `../outputs/splats/`.

## Formats

- `.ply` — standard Gaussian splat PLY (what `run_pipeline.py` produces)
- `.splat` — trimmed binary format, loads faster
- `.ksplat` — mkkellogg's compact format, best for Quest 3 bandwidth

Convert `.ply` → `.ksplat` using mkkellogg's utility
(see [mkkellogg/GaussianSplats3D](https://github.com/mkkellogg/GaussianSplats3D)
for the CLI command).

## Performance tips (Quest 3 standalone)

- **≤400k Gaussians** is the practical upper bound for 72 FPS
- Lower SH degree (0 or 1) reduces per-splat cost
- Prefer `.ksplat` over `.ply` for bandwidth
- If standalone FPS is too low, use Quest Link (tethered) for the demo
